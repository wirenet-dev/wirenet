# Sources

This is the Manager's curated Knowledge Shelf: retained evidence, source notes,
and context that should remain understandable outside one task. Treat it as
read-only by default and prefer links or small typed summaries over copying
private, large, or operational source trees.

External project workspaces remain the canonical home for code, media, data,
and working documents. `sources/` complements those bindings; it does not
duplicate or replace them.
