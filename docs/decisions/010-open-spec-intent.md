# ADR 010: Intent to Publish the Base Format as an Open Specification

Date: 2026-07-21 · Status: accepted (intent); name deferred

## Context

The Base format is small enough to specify on a page: directory = kind, four
frontmatter keys, link verbs with a registry, certification as merge. An open
spec strengthens the neutral-layer position (ADR 006): others may adopt the
format; WireNet operates it best. A spec without a running implementation,
however, is vaporware and freezes details a pilot may still change.

## Decision

Publish the Base format as an open, unbranded specification — but only after
`base-contract.md` is built and validated in the first team pilot. The name is
chosen at spec-writing time with a proper collision check. Working title: OKF
(Open Knowledge Format) — the author's prior OKF work is the format's direct
ancestor — with a known collision recorded now: "OKF" is the established
abbreviation of the Open Knowledge Foundation (okfn.org), in the same
thematic space.

## Consequences

The strategic direction is fixed and citable; no premature public commitment
to details. Naming criteria are recorded: unbranded (adopters must not feel
they are adopting a competitor's format), descriptive, searchable,
collision-free.
